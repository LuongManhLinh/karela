-Goal-
Given a text document containing Agile User Stories, technical requirements, or system descriptions, identify all entities of specific types from the text and all relationships among the identified entities. This will be used to build a Knowledge Graph to analyze dependencies and validate INVEST criteria.

-Steps-
1. Identify all entities. For each identified entity, extract the following information:
- entity_name: Name of the entity, capitalized
- entity_type: One of the following types: [STORY_KEY, ROLE, ACTION, VALUE, RESOURCE, SYSTEM]
- entity_description: Comprehensive description of the entity's attributes, goals, or technical nature
Format each entity as ("entity"<|><entity_name><|><entity_type><|><entity_description>)

2. From the entities identified in step 1, identify all pairs of (source_entity, target_entity) that are *clearly related* to each other. Focus on dependencies, shared resources, value achievements, and potential conflicts.
For each pair of related entities, extract the following information:
- source_entity: name of the source entity, as identified in step 1
- target_entity: name of the target entity, as identified in step 1
- relationship_description: explanation of how they interact (e.g., PERFORMS, ACHIEVES, AFFECTS_RESOURCE, DEPENDS_ON, CONFLICTS_WITH, PART_OF)
- relationship_strength: a numeric score indicating the strength or criticality of the relationship (1-10)
Format each relationship as ("relationship"<|><source_entity><|><target_entity><|><relationship_description><|><relationship_strength>)

3. Return output in English as a single list of all the entities and relationships identified in steps 1 and 2. Use **##** as the list delimiter.

4. When finished, output <|COMPLETE|>
